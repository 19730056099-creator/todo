# 待办事项应用 — 产品需求文档 (PRD)

## 1. 产品定位

一个轻量级的个人待办事项管理工具，帮助用户在日常工作中快速记录、追踪和回顾任务。面向个人用户，聚焦于核心的增删改查与基础统计，不做协作或多端同步。

## 2. 用户故事

| ID | 角色 | 需求 |
|---|---|---|
| US-01 | 用户 | 我想快速添加一条待办，包含标题、描述、截止日期、优先级和分类信息 |
| US-02 | 用户 | 我想查看所有待办的列表，并按分类、优先级、完成状态筛选，或通过关键词搜索 |
| US-03 | 用户 | 我可以编辑已有待办的任意字段，也可以删除不再需要的待办 |
| US-04 | 用户 | 我希望刷新页面后数据不丢失 |
| US-05 | 用户 | 我想看到整体完成率和待办数量，了解进度 |

## 3. 功能需求

### F-1 添加待办
- 表单字段：标题（必填）、描述、截止日期、优先级（高/中/低）、分类（自定义标签）
- 保存后自动关闭表单并刷新列表

### F-2 查看与筛选
- 列表展示所有待办，每项显示标题、优先级、截止日期、状态
- 筛选维度：分类、优先级、完成状态
- 搜索：按标题/描述关键词实时过滤

### F-3 编辑与删除
- 点击待办可编辑任意字段
- 删除前确认
- 待办可标记为已完成/未完成

### F-4 数据持久化
- 所有数据写入 localStorage
- 页面加载时从 localStorage 恢复

### F-5 统计面板
- 总待办数、已完成数、未完成数
- 完成率（百分比）
- 按优先级/分类的分布统计（概览级）

## 4. 非功能需求

| 维度 | 要求 |
|---|---|
| 性能 | 本地操作，无网络请求；列表渲染无感（< 100 条默认无虚拟化需求） |
| 兼容性 | 现代浏览器（Chrome/Firefox/Edge 最新两个大版本） |
| 可用性 | 空状态提示、操作反馈（添加/删除/编辑成功后有轻提示） |
| 数据安全 | 仅存 localStorage，不主动上传；用户主动清除浏览器数据会丢失 |

## 5. 技术方案摘要

| 层 | 选型 | 理由 |
|---|---|---|
| 框架 | React + TypeScript + Vite | 类型安全，开发体验好，构建快 |
| 状态管理 | Zustand | 比 Redux 轻量，API 简洁，足以覆盖本应用规模 |
| 样式 | Tailwind CSS | 原子化 CSS，快速迭代，无额外运行时 |
| 日期处理 | date-fns | 按需引入，体积小，功能满足日期格式化与比较 |
| 数据存储 | localStorage | 零后端，满足持久化需求 |
